# 🧱 SculptorTS v1 — Full Specification

> **Tagline:** Shape your backend, don’t fight it.
> **CLI:** `sc`
> **Namespace:** `@sculptor/*`

---

# 1. 🎯 Product Definition

**SculptorTS v1** is a **hybrid backend framework built on Express**, designed to provide structured architecture while preserving full access to native Express APIs.

---

## Core Capabilities

* Hybrid routing: **functional + decorator-based**
* Central **registry-driven architecture**
* Full-featured **CLI workflow**
* Config-driven runtime via `props.json`
* Framework behavior control via `sculptor.json`
* Zero lock-in: Express remains fully usable

---

## Core Philosophy

> Progressive structure. Zero restriction. Full control.

---

# 4. 🧠 Core Architecture

---

## 4.1 Entry Point (`main.ts`)

### Responsibilities

* Load config
* Initialize Express app
* Create Sculptor router
* Mount router
* Start server

---

### Example

```ts
const app = express()
const config = loadConfig()

const router = createRouter({
  controllers: registry.controllers,
  routes: registry.routes,
  prefix: config.app.prefix
})

app.use(router)

app.listen(config.app.port)
```

---

---

## 4.2 Registry System (`registry.ts`)

Central declaration of all framework components.

---

### Structure

```ts
export const registry = {
  controllers: [],
  routes: [],
  services: []
}
```

---

### Rules

* Must be declarative only
* No logic or computation
* Single source of truth

---

---

## 4.3 Router System (`@sculptor/router`)

---

### Requirements

* Must support:

  * decorator-based controllers
  * functional routes
  * hybrid usage
* Must return an Express Router
* Must not restrict native Express usage

---

---

## 4.4 Router API

```ts
createRouter({
  controllers?: Class[],
  routes?: Router[],
  prefix?: string
})
```

---

---

## 4.5 Execution Pipeline

```text
Request
 ↓
Route match (functional or decorator)
 ↓
Middleware (optional)
 ↓
Controller / handler
 ↓
Response
```

---

---

## 4.6 Handler Behavior

* Supports async functions
* Automatically sends JSON response if value returned
* Forwards errors to Express error handler

# 11. ❌ Explicit Non-Goals (v1)

---

The following are intentionally excluded:

* auto-registration
* API gateway
* service discovery
* load balancing
* advanced DI
* plugin system
* distributed systems
* caching
* tracing
